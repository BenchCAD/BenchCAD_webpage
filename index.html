Introduction

BenchCAD is the first industry-grade CAD benchmark to cover the full spectrum of parametric CAD operations. Each of the 106 part families is a top-level taxonomic category — a manufacturable part class grounded in an ISO / DIN / EN specification, not a synthetic primitive — and each family unfolds into hundreds of parametric variants that exercise a realistic construction history rather than a single extrusion. Every family was defined and reviewed by senior industry CAD experts, ensuring that geometry, tolerances, and construction sequences reflect real engineering practice rather than textbook idealizations. The registry spans 2D sketching (line, arc, circle, spline, polygon, rectangle), solid construction (extrude, revolve, sweep, loft), feature modification (fillet, chamfer, shell, draft, offset), boolean composition (union, cut, intersect), patterning (linear array, polar array, mirror), and fastener-specific features (hole, counterbore, countersink, thread / helix).

32,382 parametric variants across 106 part families, grounded in 45 ISO / DIN / EN / etc. specifications — every sample shipping ground-truth CadQuery, STEP, a multi-view composite render, and numeric QA pairs. Four parallel evaluation axes:

  • Image → CadQuery code generation
  • 3D understanding
  • Code understanding
  • Code editing

Four Parallel Tasks

Image → Code

CadQuery generation from a rendered part

Model sees a 2×2 composite render, outputs a CadQuery program. We re-execute, render, and score against GT via IoU (voxel 64³), Chamfer (2048 pts), and Feature-F1.

Image → QA

Visual numeric reasoning

Model answers numeric questions about the rendered part (hole diameter, fillet radius, counts, ratios). Scored by symmetric ratio accuracy.

Code → QA

Text-only numeric reasoning

Model reads ground-truth CadQuery source and answers the same numeric questions. Isolates code-understanding from visual perception.

Edit Code

Parametric edit from delta renders

Given original CadQuery plus a target render reflecting a 2–5% parameter delta, the model edits the code. We execute and score the edited geometry against the GT-edit via IoU / Chamfer.
